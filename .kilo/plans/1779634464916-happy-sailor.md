# Stage Unlock Redesign Plan

## Goal
Redesign the game stage mechanism so that:
- All normal stages (non-boss) are freely playable without any unlock requirement
- Boss stages require unlocking (complete all previous stages in the world to unlock the boss)

## Current Behavior
- `selectStage(worldId, stageId)` in world info panel directly starts battle
- No distinction between boss and normal stage unlock logic
- Boss stages currently don't have unlock checks

## New Behavior
1. **Non-boss stages**: Always playable, no unlock check needed
2. **Boss stages**: Require all previous stages in the world to be completed before unlocking

## Implementation Plan

### 1. Modify `selectStage()` function (line ~4406)
Add boss unlock check:
```javascript
function selectStage(worldId, stageId) {
  const world = topicConfigs[worldId];
  if (!world) return;

  const stage = world.stages.find(s => s.id === stageId);
  if (!stage) return;

  // For boss stages, check if unlocked
  if (stage.boss && !isBossUnlocked(worldId, stageId)) {
    alert('需要完成所有關卡才能挑戰Boss！');
    return;
  }

  // ... rest of function
}
```

### 2. Add `isBossUnlocked(worldId, stageId)` helper function
Check if all previous stages (non-boss) in the world are completed:
```javascript
function isBossUnlocked(worldId, stageId) {
  const world = topicConfigs[worldId];
  if (!world) return false;

  const worldProgress = JSON.parse(localStorage.getItem('worldProgress') || '{}')[worldId] || {};

  // Get all non-boss stages before this boss
  const previousStages = world.stages.filter(s =>
    !s.boss && s.id < stageId
  );

  // Check if all previous stages are completed
  return previousStages.every(s => worldProgress[s.id]?.completed);
}
```
