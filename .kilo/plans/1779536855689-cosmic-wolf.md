## 1. DESIGN PRINCIPLES

## 2. INPUT DIFFICULTY LEVELS (Level Selection)

### 2.1 Level 1: Multiple-Choice (3 Options)
| Aspect | Specification |
|--------|---------------|
| Display | 3 buttons in single row or wrapped layout |
| Distractor Range | ±15-30% from correct answer |
| Use Case | Beginners / young learners |
| Visual | Large touch targets (min 60px height) |
| Keyboard | 1, 2, 3 keys map to options |

### 2.2 Level 2: Multiple-Choice (4 Options)
| Aspect | Specification |
|--------|---------------|
| Display | 2x2 grid layout |
| Distractor Range | ±10-25% from correct answer |
| Use Case | Intermediate learners |
| Visual | Balanced grid, clear separation |
| Keyboard | 1, 2, 3, 4 keys map to options |

### 2.3 Level 3: Direct Input (Numeric Keypad)
| Aspect | Specification |
|--------|---------------|
| Display | On-screen numpad + input field |
| Input Width | 4 digits max |
| Submit | ✓ button or Enter key |
| Use Case | Advanced learners / exam prep |
| Visual | Clean input field, prominent submit |

}

### 8.2 Question Generation by Stage
```javascript
function generateBattleQuestion(level, stageId) {
  const stage = battleStages.find(s => s.id === stageId);
  const tableOptions = stage.tableRange;

  // Pick random table from stage's range
  const num1 = tableOptions[Math.floor(Math.random() * tableOptions.length)];
  const num2 = Math.floor(Math.random() * 9) + 2; // 2-10

  // Generate answer
  const answer = num1 * num2;

  // Generate options based on level
  const options = level === 3
    ? null  // Direct input, no options
    : generateOptions(answer, levelConfig[level].optionCount);

  return {
    questionText: `${num1} × ${num2} = ?`,
    answer: answer,
    options: options,
    table: num1,
    secondFactor: num2
  };
}
```

## 12. DESIGN DECISIONS (CONFIRMED)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Stage Access | **All Open** | Users can access any stage (1-6) immediately |
| Teaching Mode | **Complete Separation** | Teaching = visual learning, Battle = pure practice |
| Level Selection | Required before Stage | Users must pick input difficulty first |
| Progress Tracking | Per Level + Stage | Stars saved for each (level, stage) combination |

### 12.1 Unlocked Stage Behavior
All 6 stages are accessible from the start regardless of level selected. No sequential unlock requirement.

### 12.2 Teaching vs Battle Decoupling

**Teaching Section (Learn Tab)**:
- Visual aids: dot arrays, number lines, blocks, stories, finger counting
- No timer pressure
- Hints available on request
- Cantonese explanations
- Focus: Concept understanding and multiplication foundation

**Battle Section (Battle Tab)**:
- Pure practice: no visual aids, no hints
- Timer pressure based on selected level
- Input mode based on level (3-choice, 4-choice, direct input)
- Focus: Speed and accuracy improvement

### 13.1 Level Recommendation Logic
```javascript
function suggestLevel(userId) {
  const stats = getUserStats(userId);
  const accuracy = stats.correct / stats.total;

  if (accuracy >= 0.9) return 3;      // Direct input - high performers
  if (accuracy >= 0.7) return 2;      // 4-choice - intermediate
  return 1;                            // 3-choice - beginners
}
```

### 13.2 Star Rating (Per Stage)
Stars are awarded based on stage performance only, regardless of which level was used:
| Stars | Condition |
|-------|-----------|
| ⭐ | Complete stage |
| ⭐⭐ | Accuracy >=70% OR time >30% remaining |
| ⭐⭐⭐ | Accuracy >=90% AND time >50% remaining |

Best star rating per stage is saved. Level choice doesn't affect stars.
