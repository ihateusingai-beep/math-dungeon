# UX/UI Audit Plan: Battle Mechanism & Answer Stage
# Multiplication Town - Comprehensive Audit

**Date**: 2026-05-23
**Auditor**: Kilo Agent
**Scope**: Battle mechanism (lines 1436-2000+, 2040-2504) + Answer stage UI (2953-3003)

---

## Audit Objective

Conduct a thorough UX/UI evaluation of the battle mechanism and answer stage to identify friction points, usability issues, and improvement opportunities.

---

## 1. VISUAL HIERARCHY ANALYSIS

### 1.1 Battle Arena Layout (lines 2040-2300)
**Current Structure**:
```
[Stage Banner - width: 100%]
[Battle Header - timer, level name, score]
[Battle Attributes - shield, combo, wave]
[Battle Duel - hero | question | monster]
[Number Pad - 3x4 grid]
```

**Issues Identified**:
- Question font (3.2rem at line 2235) competes visually with monster (4rem emoji at line 1475)
- Timer bar is narrow (14px) and low contrast against dark background
- HP bars have inconsistent styling: `.hp-bar` (lines 2206-2227) vs `.hp-bar-bg` (line 1491)
- Hero/Monster HP bars lack numeric labels in duel view

### 1.2 Color & Contrast
| Element | Current | Problem |
|---------|---------|---------|
| Battle Question | `#FFD93D` (yellow) on `rgba(0,0,0,0.3)` | Good contrast but same color as accent buttons |
| Timer Text | `#4ade80` (green) | Low visibility when timer is critical |
| HP Fill States | green/yellow/red gradients | Missing transition states between phases |
| Answer Mode Toggle | Active = yellow, Inactive = transparent | Mode switching not obvious |

### 1.3 Typography
- Question text: 3.2rem (potentially too large for mobile)
- Number pad buttons: Same size as question options but 1.3rem
- Stage name: 1.1rem, Monster name: 0.9rem

## 2. INTERACTION DESIGN ANALYSIS

### 2.1 Answer Input Flow (lines 5081-5102, 5174-5201)

**Input Mode Issues**:
- `type="tel"` triggers numeric keyboard but has no clear submit affordance on mobile
- Submit button (`battle-submit-btn`) is small (70px min-width)
- Enter key works but no visual hint
- Input field border (`2px solid var(--accent)`) is subtle when focused

**Choice Mode Issues** (lines 5140-5162):
- 3 options displayed in 2-column grid (line 1529) with no visual grouping
- Correct answer has 15-30% offset range - may create confusing distractors
- No keyboard navigation support

### 2.2 Number Pad (lines 2989-3003)
```
[1] [2] [3]
[4] [5] [6]
[7] [8] [9]
[C] [0] [E]
```

**Issues**:
- C = Clear, E = Enter (not intuitive for target age group)
- No physical keyboard support when numpad is active
- Button size: not defined (uses default `.num-btn` from line 145)
- Special buttons (C/E) have `.special` class but no distinct visual treatment

### 2.3 Answer Mode Toggle (lines 2957-2960)
- Two buttons: "⌨️ 直接輸入" and "🔘 選擇題"
- Active state: yellow background, white text
- Inactive: transparent with white text at 70% opacity
- **Issue**: No indication of which mode is currently active during battle

### 2.4 Timer UX (lines 2104-2128)
- Visual bar shrinks from right to left
- Warning state at 50%: yellow gradient
- Danger state at 25%: red with pulse animation
- **Issue**: Time remaining shown as bar, no numeric countdown visible in battle header

## 3. USER FLOW ANALYSIS

### 3.1 Battle Entry Flow
```
Stage Selector → Start Battle → Battle Overlay →
Hero/Monster Appear → Question Display → Answer → Feedback → Next Question
```

**Friction Points**:
1. Stage selector (lines 3006-3015) appears AFTER clicking "開始戰鬥" - no preview
2. Battle overlay (lines 1881-1900) fades in over 0.4s but no clear "Get Ready" state
3. Questions appear immediately with no brief pause

### 3.2 Answer Submission Flow
```
Type/Choose Answer → Submit → Immediate Feedback →
(If Correct: damage number, monster shake, next question)
(If Wrong: input border turns red, correct answer shown, hero takes damage)
```

**Issues**:
- Correct answer hint appears below question (line 5296) - clutters UI
- 1500ms delay after wrong answer (line 5301) before next question - feels slow
- No progressive disclosure of combo milestones

### 3.3 Battle Exit Flow
```
Monster HP = 0 → Victory Overlay → Reward Summary → 
[Continue] [Exit to Stage Select]
```

**Issues**:
- Victory overlay (line 1877) shows rewards but no replay option
- No clear way to retry same stage for better stars

## 4. ACCESSIBILITY ISSUES

### 4.1 Keyboard Navigation
- Battle options (`.battle-option`) have no focus states defined
- Number pad has no tab order
- No skip link for battle section

### 4.2 Screen Reader
- Monster images have `alt` attribute but no aria-label for battle state
- HP bars lack aria-valuenow, aria-valuemin, aria-valuemax
- Timer not announced to screen readers

### 4.3 Visual Accessibility
- Animations: monster float, hp pulse, combo glow run infinitely
- Screen shake on wrong answer (line 2000-2001)
- No `prefers-reduced-motion` handling for battle-specific animations

### 4.4 Touch Targets
- Number pad buttons: unclear sizing (line 145 suggests 44px but not explicit for numpad)
- Battle options: 1.3rem text in 12px padding - may be too small

---

## 5. MOBILE-SPECIFIC ISSUES

### 5.1 Layout (lines 2040-2080)
- Battle arena max-width: 520px (line 1454)
- On small screens, question text (3.2rem) may overflow
- Number pad buttons may be too small for fat fingers

### 5.2 Input
- `type="tel"` doesn't guarantee numeric keyboard on all devices
- Submit button position - right of input, may be covered by keyboard

### 5.3 Viewport
- No viewport height handling for battle overlay
- Safe area insets not considered for notched devices

## 6. INFORMATION ARCHITECTURE

### 6.1 Battle State Indicators
| Indicator | Location | Update Frequency |
|-----------|----------|------------------|
| Hero HP | Battle header (lines 1550-1577) | On damage |
| Monster HP | Monster card (lines 2970-2973) | On attack |
| Combo Count | Floating (line 3023-3025) | On correct |
| Timer | Header (lines 2099-2134) | 100ms |
| Shield | Header (line 2981) | On combo >= 3 |
| Gold | Header (line 1830) | On monster defeat |

**Issue**: Shield count (line 2982) shows "0/100" - unclear if 0 shields or 100 energy needed

### 6.2 Feedback Types
- Correct: Green flash, damage number, monster shake, victory sound
- Wrong: Red flash, screen shake, correct answer reveal, hero damage
- Timeout: Gray flash, "時間到" indicator, hero damage
- Critical: Purple flash, larger damage number, enhanced effects

**Issue**: Too many simultaneous effects may overwhelm young users

---

## 8. RECOMMENDATIONS SUMMARY

### High Priority
1. Add numeric countdown to timer display
2. Improve answer mode toggle visibility during battle
3. Fix shield display text ("0/100" → "⚡ 0")
4. Add visual feedback for magic burst combo milestones

### Medium Priority
1. Redesign correct answer reveal (modal instead of inline)
2. Improve numpad button labels ("C" → "✕", "E" → "✓")
3. Add stage preview before entering battle
4. Increase monster HP text size
5. Add keyboard navigation for battle options

### Low Priority
1. Reduce simultaneous animation effects
2. Add prefers-reduced-motion for battle animations
3. Add retry option on victory screen
4. Improve question type badge visibility

---

## Implementation Approach

Given the scope, I recommend addressing issues in this order:

1. **Quick Fixes** (Visual/UX): Timer numeric, shield text, numpad labels
2. **Interaction Fixes**: Answer mode visibility, correct answer reveal, keyboard nav
3. **Polish**: Animation consolidation, stage preview, retry option

**Estimated Complexity**: Medium (multiple UI locations, affects battle flow)
