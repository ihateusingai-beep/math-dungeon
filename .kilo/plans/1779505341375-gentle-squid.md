# UI Scaling Issue Fix Plan - Battle Stage Selection

## Problem Statement
After selecting a level combat stage, the combat interface does not display in full screen and parts of the UI are being cut off.

## Root Cause Analysis

### 1. Layout Container Structure
- `.main-section` has `max-width: 600px` (line ~2350)
- `.battle-section` has `max-width: 600px` inherited (line ~1437)
- `.battle-arena` has `max-width: 520px` (line 2748)
- `.stage-selector` has `max-width: 520px` (line 2849)
- `.stage-banner` has no explicit max-width, inherits from parent

### 2. CSS Hierarchy Issue
When stage is selected:
1. `selectStage()` calls `closeStageSelector()` which sets `display: none` via inline style
2. `stage-banner` is shown with `display: block`
3. The stage banner uses `width: 100%` but may overflow due to border-box sizing

### 3. Suspected Issues
- Stage banner missing explicit `max-width` constraint
- Stage selector uses inline styles that may conflict with CSS
- The `.battle-header` has `width: 100%` which may cause overflow when combined with padding/borders
- No responsive breakpoints for smaller screens

## Proposed Fix

### Fix 1: Add max-width to stage-banner
```css
.stage-banner {
  width: 100%;
  max-width: 520px;  /* Add this to match arena */
  ...
}
```

### Fix 2: Ensure consistent box-sizing
```css
.stage-banner, .battle-header {
  box-sizing: border-box;
}
```

### Fix 3: Add responsive breakpoints for mobile
```css
@media (max-width: 520px) {
  .battle-arena, .stage-selector, .stage-banner {
    max-width: 100%;
    padding: 10px;
  }
  .battle-header {
    flex-wrap: wrap;
    gap: 8px;
  }
  .timer-bar {
    width: 120px;
  }
}
```

### Fix 4: Remove inline style conflict on stage-selector
Change `closeStageSelector()` to toggle a class instead of inline style:
```javascript
function closeStageSelector() {
  document.getElementById('stageSelector').classList.remove('active');
}
```
With CSS:
```css
.stage-selector {
  display: none;
}
.stage-selector.active {
  display: block;
}
```

## Implementation Steps

1. **Add CSS for stage-banner max-width and box-sizing** (around line 2029)
2. **Add responsive media queries** for mobile/small screens
3. **Refactor stage-selector to use class toggle** instead of inline styles
4. **Test on mobile viewport** to verify fix
