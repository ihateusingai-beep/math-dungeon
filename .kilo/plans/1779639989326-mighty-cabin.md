## 🎯 Remaining Work — Detailed Implementation Plan

### Phase 3.5: Client-Side State Sync & WebSocket Integration

#### Why this matters
The server skeleton accepts connections but doesn't integrate with GameEngine. We need a `NetworkClient` that:
1. Connects client GameEngine to WebSocket server
2. Sends actions to server and receives validated state
3. Reconciles client state with server authoritative state
4. Handles late-join spectators

#### Architecture: `NetworkClient` Class

```javascript
// src/engine/NetworkClient.js
class NetworkClient {
  constructor(gameEngine, options = {}) {
    this.engine = gameEngine;
    this.ws = null;
    this.playerId = null;
    this.roomId = null;
    this.pendingActions = new Map();  // actionId -> resolve
    this.serverState = null;
    this.stateHashInterval = null;
    this.onRemoteStateUpdate = options.onRemoteStateUpdate || null;
  }

  connect(serverUrl) { /* WebSocket connection */ }
  createRoom(hostName, heroKey) { /* sends CREATE_ROOM */ }
  joinRoom(roomId, playerName, heroKey) { /* sends JOIN_ROOM */ }
  sendAction(action, data) { /* sends PLAYER_ACTION, returns Promise */ }
  requestFullStateSync() { /* sends REQUEST_STATE */ }
  
  // Internal
  handleMessage(message) { /* route incoming messages */ }
  reconcileState(serverState) { /* compare hashes, rollback if needed */ }
  computeStateHash(state) { /* MD5-like hash for desync detection */ }
}
```

#### State Reconciliation Flow

```
Client Action                    Server Processing
    │                                │
    ├─ sendAction(answer) ──────────►│ validate answer
    │                                │ update server state
    │                                │ compute new state hash
    │◄──────── ACTION_RESULT ────────┤
    │                                │
    │   (render optimistically)      │
    │                                │
    │◄──────── STATE_UPDATE ─────────┤ (periodic or on demand)
    │   (state + hash)               │
    │                                │
    ├─ compare hashes               │
    │   └─ match? ──── continue     │
    │   └─ mismatch? ── request     │
    │       full sync               │
```

#### State Hash (Desync Detection)

```javascript
// Simple hash for state comparison
computeStateHash(state) {
  const str = JSON.stringify({
    phase: state.phase,
    heroHp: state.heroHp,
    monsterHp: state.monsterHp,
    combo: state.combo,
    gold: state.gold
  });
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    const char = str.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash = hash & hash;
  }
  return hash.toString(16);
}
```

#### Late-Join Spectator Mode

When a player joins mid-battle:
1. Server sends `STATE_SYNC` with current battle state
2. Client receives and calls `engine.loadState(state)`
3. Client renders but disables action buttons
4. On next turn boundary, client re-enables actions

rollHp(playerCount = 1) {

### Phase 4: Polish & Anti-Cheat

#### 4.1 Performance Optimization

| Item | Target | Implementation |
|------|--------|----------------|
| Render loop | 60fps | Use `requestAnimationFrame` with dirty checking |
| DOM batch updates | <16ms | Batch reads/writes, double buffering |
| Image lazy load | — | `IntersectionObserver` for off-screen images |
| State updates | <5ms | Avoid deep clones, use delta updates |

**Dirty Checking Pattern:**
```javascript
let lastStateHash = null;
function renderLoop() {
  const state = engine.getState();
  const hash = computeStateHash(state);
  if (hash !== lastStateHash) {
    updateUI(state);
    lastStateHash = hash;
  }
  requestAnimationFrame(renderLoop);
}
```

#### 4.2 Anti-Cheat Light

**Server-side answer validation (already planned):**

In `server.js` `processAnswer()`:
```javascript
function processAnswer(data, room) {
  const { input, questionId } = data;
  const state = room.state;
  
  if (!state.currentQuestion) {
    return { success: false, message: 'No active question' };
  }
  
  // Server validates - client cannot be trusted
  const correct = parseInt(input) === state.currentQuestion.answer;
  
  if (!correct) {
    // Apply monster attack to shared HP pool
    room.sharedHp = (room.sharedHp || 100) - 10;
  }
  
  return { success: true, correct, timestamp: Date.now() };
}
```

**Rate Limiting:**

```javascript
// In server.js
const playerActionTimes = new Map();  // playerId -> last action timestamp

function rateLimit(playerId, actionType) {
  const now = Date.now();
  const lastAction = playerActionTimes.get(playerId) || 0;
  
  if (now - lastAction < 1000) {  // 1 second cooldown
    return { allowed: false, reason: 'Rate limited (1/sec)' };
  }
  
  playerActionTimes.set(playerId, now);
  return { allowed: true };
}
```

#### 4.4 Cloud Save Sync

**API Design:**
```javascript
// Save to cloud
POST /api/save
{ playerId, heroState, progressState, timestamp }

// Load from cloud
GET /api/load/:playerId

// Conflict resolution: server timestamp wins
```

**Implementation:**
```javascript
// In SaveManager
async saveToCloud(playerId) {
  const data = this.load();
  const response = await fetch('/api/save', {
    method: 'POST',
    body: JSON.stringify({ playerId, ...data })
  });
  return response.json();
}

async loadFromCloud(playerId) {
  const response = await fetch(`/api/load/${playerId}`);
  const data = await response.json();
  
  // Compare timestamps
  const local = this.load();
  if (data.timestamp > local.timestamp) {
    this.importSave(JSON.stringify(data));
    return { source: 'cloud', data };
  }
  return { source: 'local', data: local };
}
```
