# Separate Battle Stage from Teaching Tools

## Goal
Decouple the battle/stage game mode from the teaching/learning tools by adding a dedicated "Game" section.

## Selected Approach: Option B - Four-Tab Model

Navigation tabs will be:
- **Learn (學習)** - Pure teaching tools only (no battle elements)
- **Battle (戰鬥)** - Keep existing battle section for quick access
- **Game (遊戲)** - NEW: Stage Map + Hero Management + Rewards + Inventory
- **Profile (我的)** - User profile (existing)

## Implementation Steps

### Step 1: Create New Game Section
Add new `gameSection` HTML containing:
- Hero Management (level, evolution, stats)
- Inventory system (gold, potions, scrolls)
- Stage Map access
- Rewards/Loot display
- Daily rewards

Location: After `battleSection` (line ~3121), before `profileSection`

### Step 2: Update Navigation Bar
Find nav buttons (line ~2576-2578) and add Game tab:
```html
<button class="main-nav-btn" data-section="game" onclick="switchMainSection('game')">🎮 遊戲</button>
```

### Step 3: Clean Up Learn Section
Remove from `learnSection`:
- Hero team display (lines 2607-2626) - move to Game section
- "⚔️ 開始戰鬥" button (line 2691) - keep for quick battle access or remove
- "🗺️ 關卡地圖" button (line 2749) - move to Game section

### Step 4: Create Hero Management UI
In new `gameSection`:
- Hero cards showing current hero stats
- Level/XP progress bar
- Evolution stage display
- Skill tree preview
- Equipment slots

### Step 5: Create Inventory UI
In new `gameSection`:
- Gold display
- Item inventory (potions, scrolls, fragments)
- Loot collection

### Step 6: Update JavaScript
Add `switchMainSection('game')` handling:
- Load hero data
- Display inventory
- Show stage map button

### Step 7: State Independence
Ensure `battleState` and teaching state are separate:
- Teaching state → `multiplicationData` (localStorage)
- Battle state → `battleData` (localStorage)
- Hero/Game state → `mathTownHero` + new `gameInventory` (localStorage)
