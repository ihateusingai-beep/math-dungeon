# Plan: 開放所有關卡選擇及選擇關卡後才開始戰鬥

## 問題分析

1. **關卡未全部開放**: 目前 stages 陣列中，只有關卡 1 的 `unlocked: true`，其他關卡需要完成前置關卡才能解鎖
2. **開始戰鬥按鈕直接開始戰鬥**: 用戶按下「開始戰鬥」時沒有先選擇關卡，而是直接使用當前設定的 stage

## 修改方案

### 3. 開放所有關卡 (index.html ~5621-5631)

### 4. 確保載入進度時不覆蓋解鎖狀態 (index.html ~5681-5688)

現有 `loadStageProgress()` 邏輯:
```javascript
stage.unlocked = data.unlocked || stage.unlocked;
```

這個邏輯是正確的 — 如果 localStorage 中沒有某關卡的存檔，會使用 stages 陣列中的預設值（我們設的 `true`）。所以不需要修改這裡。

## 預期結果

1. **所有 9 個關卡預設為可選擇狀態**（`unlocked: true`）
2. 用戶點擊「開始戰鬥」按鈕時：
   - 若未選擇關卡 → 顯示關卡選擇器
   - 若已選擇關卡 → 直接開始戰鬥
3. 用戶在關卡選擇器中選擇關卡後，**自動開始戰鬥**（不需再點一次開始）
4. 戰鬥中的敵人根據選擇的關卡生成

### 2. 修改「開始戰鬥」按鈕行為 (index.html ~4908)

**目標流程**:
1. 用戶點擊「開始戰鬥」按鈕
2. 顯示關卡選擇器 (stageSelector)
3. 用戶選擇關卡後，自動開始戰鬥

**修改 `startBattle()` 函數**:

```javascript
function startBattle() {
  // 如果未選擇關卡，先顯示關卡選擇器讓用戶選擇
  if (!battleState.currentStage) {
    // 顯示關卡選擇器，選擇後自動開始
    showStageSelectorForBattle();
    return;
  }
  // 如果已有選擇的關卡，直接開始戰鬥
  startBattleWithStage();
}

function showStageSelectorForBattle() {
  // 隱藏其他界面，顯示關卡選擇器
  document.getElementById('levelSelectScreen').style.display = 'none';
  document.getElementById('stageSelector').style.display = 'block';
  document.getElementById('battleStartBtnOverlay').style.display = 'none';
  renderStageSelector();
}

function startBattleWithStage() {
  // 原有 startBattle 的邏輯...
}
```

**修改 `selectStage()` 函數**:
選擇關卡後自動開始戰鬥，而不是顯示「開始戰鬥」按鈕:

```javascript
function selectStage(stageId) {
  const stage = getStageData(stageId);
  if (!stage.unlocked) return;

  battleState.currentStage = stageId;
  currentDomain = 'newbie';
  selectedLevel = null;

  closeStageSelector();
  // 不要顯示 battleStartBtnOverlay，直接開始戰鬥
  startBattleWithStage();
}
```

### 3. 確保關卡 ID 正確傳遞

當用戶在關卡選擇器中選擇關卡時，`selectStage(stageId)` 會設置 `battleState.currentStage = stageId`。確保這個值在 `startBattle()` 中被正確使用。
