# MathBattle 宏觀架構規劃

## 用戶選擇
- **實作範疇**: 全面規劃 (先生成完整架構，逐步實現各範疇)
- **開發方式**: 增量擴展 (保留現有乘法系統，逐步添加新主題)
- **關卡結構**: 線性世界 (完成世界內所有關卡後解鎖下一世界)

## 目標
將當前專注於「乘法」的戰鬥系統，擴展為一個涵蓋小學數學全部範疇的宏觀數學遊戲平台。

## 現有系統分析
- 當前：基於 `index.html` 的單一 HTML 架構
- 戰鬥系統：`battleSection` 使用乘法題目戰鬥
- 主頁：`rootSection` 已具備主題選擇框架（但只有乘法）
- 可擴展點：`selectTopic()` 函數預留了 `topicConfigs` 對象

## 戰鬥系統擴展設計

### 問題生成器架構
```javascript
class QuestionGenerator {
  static generate(topic, subTopic, difficulty) {
    switch(topic) {
      case 'multiplication':
        return this.generateMultiplication(difficulty);
      case 'addition':
        return this.generateAddition(difficulty);
      case 'subtraction':
        return this.generateSubtraction(difficulty);
      case 'division':
        return this.generateDivision(difficulty);
      case 'comparison':
        return this.generateComparison(difficulty);
      case 'patterns':
        return this.generatePattern(difficulty);
      case 'fractions':
        return this.generateFraction(difficulty);
      // ... 其他主題
    }
  }
}
```

## 總結

這個架構將把 MathBattle 從單一乘法訓練工具，轉變為覆蓋小學數學全部範疇的完整遊戲平台。採用世界觀+主題的雙層架構，既保持了遊戲的趣味性，又確保了教育的系統性。

核心原則：
1. **可擴展性** - 新主題易於添加
2. **一致性** - 統一用戶體驗
3. **系統性** - 覆蓋完整數學範疇
4. **遊戲化** - 保持學習動機
