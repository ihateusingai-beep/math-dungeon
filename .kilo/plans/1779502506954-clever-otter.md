# Battle Stage UI/UX Design Recommendations

## Context
This plan documents comprehensive UI/UX design recommendations for a video game battle stage, specifically tailored for an educational multiplication game (乘法小鎮 / Multiplication Town). The battle system involves turn-based or timed-answer combat where players answer math questions to attack enemies.

---

## 1. Layout Recommendations

### 1.1 Screen Zones
| Zone | Position | Content |
|------|----------|---------|
| **Top Bar** | Fixed top | Level info, timer, combo counter, score |
| **Main Arena** | Center 60% | Battle scene with hero, monster, visual effects |
| **Bottom Bar** | Fixed bottom | Answer input area (number pad or choices) |
| **Side Panels** | Left/Right edges | Hero stats (HP bar, level), Enemy HP |

### 1.2 Visual Hierarchy
- **Primary focal point**: Hero and monster sprites in center
- **Secondary**: Question/answer area at bottom (player action zone)
- **Tertiary**: Combo counter, timer (peripheral awareness)

### 1.3 Responsive Layout
- **Desktop**: Wider arena, side-by-side hero/enemy HP bars
- **Tablet**: Stacked layout, larger touch targets (44px minimum)
- **Mobile**: Full-width arena, bottom-anchored answer panel, collapsible stats

## 2. Essential HUD Elements

### 2.1 Health Bars
| Element | Design Recommendation |
|---------|---------------------|
| **Hero HP** | Left side, vertical or curved bar, green→yellow→red gradient as HP drops |
| **Enemy HP** | Above enemy sprite or right side, same color coding |
| **HP Numbers** | Always visible: "120 / 150 HP" format |
| **Damage Flash** | Red flash + shake on HP bar when taking damage |
| **Heal Effect** | Green pulse + floating "+HP" number |

### 2.2 Mana/Stamina (if applicable)
- Use blue/purple gradient for mana bars
- Partial fill indicators for abilities on cooldown
- Energy/stamina for special attacks shown as segmented bar

### 2.3 Cooldown Timers
- **Radial cooldown**: Circular icon overlay showing time remaining (0-100%)
- **Numerical display**: Small seconds counter beside ability icon
- **Glow pulse**: Ability icon pulses when ready
- **Desaturated**: Grey out abilities during cooldown

### 2.4 Battle Timer
- **Position**: Top center or near question area
- **Visual**: Animated bar that depletes left-to-right
- **Color coding**: Green (>50%) → Yellow (25-50%) → Red (<25%)
- **Urgency effects**: Pulse/shake when <25% time remaining

### 2.5 Combo Counter
- **Position**: Top-right or below timer
- **Display**: Large number with "COMBO" label
- **Visual feedback**: Scale-up animation on increment, fire/spark effects at thresholds (5, 10, 15...)
- **Combo decay**: Visual fade when combo resets

## 3. Combat Feedback Mechanisms

### 3.1 Damage Numbers
| Type | Style |
|------|-------|
| **Normal damage** | White/yellow numbers, float upward and fade |
| **Critical hit** | Larger, orange/red, "CRIT!" label, screen shake |
| **Weak hit** | Smaller, grey, slower animation |
| **Miss/Blocked** | "MISS" or "BLOCKED" in grey, no upward float |
| **Combo damage** | Stacked numbers showing cumulative damage |

**Animation specs:**
- Duration: 800-1200ms
- Movement: Float up 50-80px
- Scale: Start at 1.2x, end at 0.8x
- Opacity: Fade out in last 300ms

### 3.2 Hit Effects
- **Screen shake**: 3-5px displacement, 100-200ms duration, intensity scales with damage
- **Flash overlay**: Brief white/red tint on taking damage
- **Hit spark**: Particle burst at point of impact
- **Elemental effects**: Color-coded particles (fire=orange, ice=blue, etc.)

### 3.3 Death/Defeat Effects
- Monster fade out + shrink
- Explosion particle effect
- "VICTORY" banner with fanfare
- Gold/exp reward popup

### 3.4 Level Up/Evolution Effects
- Golden glow radiating from hero
- Star burst particle effect
- Brief pause in gameplay (100-200ms) for impact
- "+20 HP" stat change indicators

## 4. Immersion vs. Legibility Balance

### 4.1 Color Palette Guidelines
| Purpose | Recommendation |
|---------|----------------|
| **Background** | Dark, atmospheric but not distracting (semi-transparent overlays) |
| **UI Chrome** | Semi-transparent dark panels (rgba(0,0,0,0.6)) with subtle borders |
| **Critical info** | High contrast - use bright yellow (#FFD93D), green (#6BCB77), red (#FF6B6B) |
| **Text** | White text on dark backgrounds, avoid pure black |
| **Accessibility** | Support colorblind modes, ensure 4.5:1 contrast ratio minimum |

### 4.2 Information Hierarchy by Priority
1. **Must see**: Question being asked, answer input area, your HP
2. **Should see**: Enemy HP, timer, combo count
3. **Nice to see**: Background details, decorative effects
4. **Can hide**: Low-priority UI that can be toggled

### 4.3 Visual Noise Management
- **Depth of field**: Blur distant/non-essential UI elements
- **Vignette**: Subtle darkening at screen edges to focus attention center
- **Pause non-critical animations**: When intense action occurs, simplify background effects
- **Clean combat moments**: Between attacks, show full UI; during attacks, minimize clutter

### 4.4 Typography
- **Question text**: Large (2-3rem), bold, high contrast
- **Stats/numbers**: Monospace or tabular figures for alignment
- **Labels**: Small but legible (14-16px minimum)
- **Font choice**: Game-appropriate but readable (avoid overly decorative fonts for critical info)

## 5. Additional Recommendations

### 5.1 Animation Timing
| Animation | Duration | Purpose |
|-----------|----------|---------|
| HP bar change | 300-400ms | Smooth but responsive |
| Damage numbers | 800-1200ms | Readable yet not obstructing |
| Screen shake | 100-200ms | Impact without disorientation |
| UI transitions | 200-300ms | Snappy but not jarring |
| Combo milestone | 400-600ms | Celebratory but brief |

### 5.2 Sound & Haptics Sync
- Hit effects should have corresponding sound cues
- Timer urgency can use audio pitch increase
- Critical moments (level up, boss defeat) deserve distinctive audio

### 5.3 Mobile-Specific
- **Touch targets**: Minimum 44x44px for all interactive elements
- **Thumb zones**: Place frequent actions in comfortable reach areas
- **Input methods**: Support both number pad and multiple-choice for accessibility
- **Orientation**: Optimize for both portrait and landscape

## 8. Design Direction (Based on User Feedback)

| Factor | Decision |
|--------|----------|
| **Target Platform** | Mobile-first with touch-optimized controls |
| **Combat Type** | Mixed (real-time with varying speed challenges) |
| **Ability Complexity** | Full feature set for engaged players |

### 8.1 Mobile-First Considerations
- **Touch targets**: Minimum 44x44px for all interactive elements (buttons, answer options)
- **Thumb zones**: Place answer input in bottom 40% of screen (comfortable reach)
- **Orientation**: Optimize for portrait mode first, landscape as secondary
- **Gestures**: Swipe to access menus, tap for confirmations
- **Number pad**: Large, spaced buttons (60px+) for accurate touch input

### 8.2 Mixed Combat Considerations
- **Dynamic timer**: Adjusts difficulty based on adaptive difficulty system (2.0s - 4.5s)
- **Question types**: Mix of speed-based (normal) and strategic (choice-based) questions
- **Combo importance**: High combos unlock bonus attacks/special abilities
- **Boss battles**: Extended time, multiple phases, more complex patterns

### 8.3 Full Complexity Features
- **Multiple ability types**: Normal attack, special abilities, ultimate skills
- **Hero passives**: Block, CritBoost, Rage (HP<30% triggers)
- **Evolution system**: Every 5 levels triggers evolution with visual fanfare
- **Adaptive difficulty**: Reactive accuracy tracking adjusts question timing
- **Combo system**: Cumulative hits with visual escalation (normal → high → max)
